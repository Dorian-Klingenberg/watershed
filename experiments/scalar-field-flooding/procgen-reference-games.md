# Procgen Reference Games

## Purpose

This note captures a small set of useful reference games for the wave function
collapse and systems-driven encounter experiments.

The purpose is not to imitate these games directly.
The purpose is to study what they do well and selectively borrow techniques
that support this project's identity:

- ancient infrastructure
- partial understanding
- intervention under uncertainty
- local action with downstream consequence

## Reference Lens

When studying any reference game, ask:

- how does the player read the current state?
- how does the game signal likely consequences?
- how many moves ahead can the player reason?
- what makes failure feel fair?
- what keeps experimentation from becoming noise?

## Hydroneer

### What It Is Useful For

- readable physical causality
- tactile intervention
- tweak-observe-refine experimentation loops
- machine layouts that become memorable through function

### What To Borrow

- visible cause-and-effect relationships
- systems that reward curiosity
- satisfying feedback from changing material flows
- interaction patterns that make players want to test hypotheses

### What To Avoid Borrowing

- drifting into factory optimization as the main fantasy
- making the game about production throughput
- overemphasizing player-built machinery
- solving problems mainly by adding more parts

### Relevance To This Project

Hydroneer is most useful as a reference for feel and readability.
It is less useful as a direct structural model.

In this project, the player should feel like they are negotiating with damaged,
inherited systems rather than building clean industrial pipelines from scratch.

## Into The Breach

### What It Is Useful For

- high puzzle legibility
- forecastable consequence
- compact but meaningful decision spaces
- fair failure through visible threat information

### What To Borrow

- clarity of consequence
- highly readable board states
- puzzle structure that supports deliberate reasoning
- pressure without hidden randomness inside the turn

### Relevance To This Project

This is one of the strongest references for solvable encounter design.
It is especially useful for thinking about how to keep generated encounters
analyzable and fair.

## Opus Magnum

### What It Is Useful For

- elegant constraint spaces
- multiple valid solutions
- satisfying experimentation inside a strict rule system

### What To Borrow

- puzzle spaces with many valid approaches
- compositional systems that stay legible
- the feeling that refinement and insight matter more than brute force

### Relevance To This Project

This is a strong reference for the abstract solvability layer above the
simulation.
It helps frame encounters as spaces of valid interventions rather than single
answer puzzles.

## Dwarf Fortress

### What It Is Useful For

- cascading systemic interactions
- unintended consequences
- environment and simulation producing stories

### What To Borrow

- the value of emergent outcomes
- local interventions causing wider effects
- the sense that systems are richer than first appearances

### What To Avoid Borrowing

- overwhelming simulation opacity
- excessive complexity before the player has a usable mental model

### Relevance To This Project

Use Dwarf Fortress as inspiration for systemic consequence, but not as a model
for interface, complexity density, or raw simulation breadth.

## Noita

### What It Is Useful For

- material interaction
- propagation behavior
- surprising chain reactions

### What To Borrow

- the idea that local material conditions matter
- visually expressive propagation
- delight from interacting systems

### What To Avoid Borrowing

- chaos that overwhelms player reasoning
- outcomes that feel too volatile to forecast

### Relevance To This Project

Noita is useful for thinking about field propagation and material states, but
the project should remain much more legible and less chaotic.

## Outer Wilds

### What It Is Useful For

- knowledge-driven progression
- environmental understanding
- discovery through observation

### What To Borrow

- progression through learning rather than stats
- trust in player interpretation
- worlds that reveal structure over time

### Relevance To This Project

Outer Wilds is a strong reference for the fantasy of understanding a system that
already exists, especially one shaped by lost builders and partial clues.

## Practical Takeaways

The likely strongest references by design need are:

- `Into the Breach` for encounter solvability and consequence legibility
- `Opus Magnum` for multi-solution puzzle structure
- `Hydroneer` for tactile systems feel and readable intervention
- `Outer Wilds` for knowledge-first discovery

Secondary references:

- `Dwarf Fortress` for cascading systemic consequences
- `Noita` for propagation and material interaction

## Summary

These reference games are useful if they are treated as targeted lenses rather
than templates.

The project's identity should remain distinct:

- not factory optimization
- not pure simulation chaos
- not scripted puzzle rooms

The target remains:

- damaged ancient infrastructure
- local repair under uncertainty
- evolving systemic encounters
- partial understanding earned through play
