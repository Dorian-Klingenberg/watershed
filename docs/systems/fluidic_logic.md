# Fluidic Logic

## Canonical Role

Ancient infrastructure should be treated as passive fluidic computation.

Preferred mechanisms:

- gravity-driven flow
- pressure head
- siphons
- threshold lips
- overflow paths
- air traps
- basin storage as memory
- geometry-biased channels
- delay basins
- oscillation chambers

Key framing:

- water and terrain compute behavior
- landscapes compute with water

For sea-lift systems, the preferred pattern is:

- slow charge
- siphon-triggered rapid discharge
- staged packet lift
- vented isolation between gains

## Design Rule

Moving parts are rare, unreliable over immense timescales, and non-essential to the core operation of ancient systems.

Avoid default assumptions of:

- valves
- hinges
- pistons
- sliding gates
- mechanical actuators

Avoid designs that accidentally turn packet systems into one giant continuous lift column.
